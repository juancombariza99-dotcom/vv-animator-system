# Skill: Ideate Visuals

## Purpose
Given a theme, transcript, script snippet, or video message, propose 2–3 visual concepts in the VV/minimal style. Each concept is a scene flow — a sequence of shapes, conditions, and actions. The user picks one to generate.

---

## Trigger phrases
- "ideate visuals for"
- "what could the animation look like for"
- "give me visual concepts for"
- "turn this into a VV-style animation"
- "what shapes and flow would work for"

---

## Steps

### 1. Read the core reference
Always start by reading the core shape vocabulary:
```
references/_core.md
```
This tells you which archetypal shapes and movements are available without loading images.

### 2. Understand the message
Extract from the user's input:
- **The core idea** — what is the single insight being communicated?
- **The conditions** — what are the "if/then" or "A vs B" or "before/after" states?
- **The action** — what should the viewer do or understand?

VV animations work best with 2–4 scenes. Each scene = one condition + one shape + one action label.

### 3. Map to visual grammar
Use this vocabulary:

**The ball** = the protagonist (mind, person, idea)
- Hollow ring = unfulfilled, searching, potential
- Solid white = formed, active, present
- Solid + glow = powerful, landed, complete

**Shapes** = contexts or tools
- Book = reading, consuming, learning
- Rectangle = page, document, writing, structure
- Triangle = leverage, hierarchy, clarity, direction
- Line = baseline, foundation, constraint
- Circle/oval = thought, idea, container

**Transitions**
- Ball falls INTO a shape = entering a context
- Shape fills solid THEN clears to outline = something blocked then revealed
- Shape morphs (rect → oval) = idea being shaped/refined
- Ball slides across a shape = passing through, page turn
- Shape draws on via dashoffset = something being created in real time

### 4. Propose 2–3 concepts
For each concept, write a brief scene flow:

```
### Concept 1 — "[title]"
Scene A: "[condition text]" — [shape] — [ball behavior] — label: "[action]"
Scene B: "[condition text]" — [shape] — [ball behavior] — label: "[action]"  
Scene C: "[condition text]" — [shape] — [ball behavior]
Transition feel: [fast/slow, sharp/soft]
Closest reference: [shape name from _core.md]
```

Keep each concept to 3–4 scenes max. One idea per scene. One shape per scene.

### 5. Ask the user to choose
Present the concepts clearly and ask which one to generate, or if they want to mix elements from different concepts.

---

## Design principles to follow
- **Minimal**: one shape, one ball, one text line per scene. No clutter.
- **Sequential**: each scene answers or continues the previous one.
- **The ball always has a clear role**: it arrives, interacts, and moves on.
- **Text is a condition, the shape is the answer, the label is the action.**
- Hold times > transition times. Let the idea breathe.
- Black background, white elements only. No color, no gradients.
